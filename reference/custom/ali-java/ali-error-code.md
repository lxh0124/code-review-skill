## 错误码格式

- 类型：**字符串**，固定 **5 位**：`来源(1位) + 编号(4位)`
- 来源：**A** 用户端（参数、版本、超时）· **B** 当前系统 · **C** 第三方
- 编号 0001–9999，大类步长预留 100；**不与**业务/组织架构挂钩，平台审批后固定
- 正常但需占位时返回 `00000`
- 禁止在错误码中编码版本号或错误等级

## 使用规则

- 原则：快速溯源、沟通标准化（谁的错、错在哪、团队认知一致）
- **禁止**随意新增错误码，优先复用附表现有码
- **禁止**把错误码直接当用户提示；须区分：`error_code` / `error_message`（排查）/ `user_tip`（用户）/ `stack_trace`（日志）
- 第三方错误可转义 B 类，并保留原第三方错误码（推荐）
- `error_message` 承载业务细节，勿让错误码本身过于臃肿（推荐）

## 与 API 联动

前后端错误响应须含 HTTP 状态码、`errorCode`、`errorMessage`、用户提示；详见 [ali-api.md](ali-api.md)。
