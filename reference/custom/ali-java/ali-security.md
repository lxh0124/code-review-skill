## 权限与数据

- 用户个人页面/功能必须做权限校验（防水平越权：查看/修改他人订单、私信等）
- 敏感数据禁止明文展示，必须脱敏（如手机号 `158****9119`）
- 用户输入 SQL 参数必须参数绑定或元数据限定，**禁止**字符串拼接 SQL

## 输入校验

- 用户传入的**任何**参数必须做有效性校验，防：
  - pageSize 过大导致 OOM
  - 恶意 `order by` 慢查询
  - 任意重定向、SQL 注入、反序列化注入、ReDoS（注意危险正则）

## Web 安全

- 禁止未经安全过滤/转义将用户数据输出到 HTML
- 表单与 AJAX 提交必须做 CSRF 校验

## 防刷与风控

- 短信/邮件/电话/下单/支付等平台资源须防重放：次数限制、疲劳度、验证码等
- 用户生成内容（发帖、评论、IM）须防刷及违禁词过滤（推荐）
